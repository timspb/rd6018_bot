# История Проекта Для Ассистента

Этот файл нужен как короткая "оперативная память" по уже принятым решениям.

## Текущее состояние (2026-03-07)

- Репозиторий и хост синхронизированы по коммиту `c71b2594aced1cfacdc642f4583103f34ca77526`.
- Основная логика заряда сосредоточена в `charge_logic.py`.
- Интерфейс Telegram и дашборд сосредоточены в `bot.py`.
- AI-часть разделена:
  - кнопочный анализ: `ai_engine.py`
  - системные правила/промпт: `ai_system_prompt.py`

## Текущая опора по стратегии (2026-05-02)

- Добавлен отдельный документ-опора `docs/assistant/CHARGE_STRATEGY.md`.
- В нём зафиксированы:
  - цепочка этапов;
  - профильные правила Ca/Ca, EFB, AGM, Custom;
  - смысл `temp_ext` и `temp_int`;
  - правила Mix, Desulfation и SAFE_WAIT;
  - граница между батарейной безопасностью и защитой железа.
- При правках FSM, AI или логов сначала сверяться с этим документом, а не вспоминать правила по памяти.

## Важные подтвержденные правила

- Глобальный лимит тока на всех этапах: `12.0A` (`MAX_STAGE_CURRENT`).
- OVP/OCP: `target + 0.1`.
- Для профилей Ca/Ca и EFB при тайм-ауте Main допустим принудительный переход в Mix.
- Дашборд должен работать как одно "рабочее" сообщение (редактирование/замена, без накопления дублей).
- `/help` не должен уходить в LLM/DeepSeek.
