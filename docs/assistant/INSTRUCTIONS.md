# Инструкции Для Ассистента В Этом Репозитории

## Цель

Держать код, документацию и хост в согласованном состоянии без расхождений.

## Рабочий порядок

1. Перед изменениями:
- проверить `git status --short --branch`;
- при рисковых правках сверить текущий `HEAD` и `origin/main`.

2. После изменений:
- запускать релевантную проверку (тест/smoke), если это код;
- если правки только в документации, явно фиксировать это в отчете.

3. Синхронизация:
- локально: коммит в `main`;
- GitHub: `git push origin main`;
- хост: `git pull --ff-only origin main`.

4. Контроль синхронизации:
- локальный `HEAD`;
- `origin/main`;
- `/root/rd6018_bot` на хосте.
Все три должны совпадать.

## Правила по интерфейсу и UX

- Не плодить новые сообщения дашборда без необходимости.
- При изменениях клавиатуры/дашборда учитывать мобильную ширину Telegram.
- Текст `/help` держать коротким, практичным, без ухода в LLM.

## Правила по AI

- Не использовать внутренние названия полей как пользовательский текст.
- Не путать защитные таймеры и "прогноз до конца этапа".
- Не придумывать данные (например, емкость), если их нет во входном контексте.
- Перед изменениями или ответами по стратегии заряда сначала открыть [CHARGE_STRATEGY.md](./CHARGE_STRATEGY.md) и сверять спорные места с ним.
- Если вопрос про температуру, помнить: `temp_ext` = АКБ, `temp_int` = БП/контроллер.
